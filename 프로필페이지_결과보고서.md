## 2. 프로젝트 요약 및 목표

**어떤 문제를 해결하기 위해 이 웹 프로그램을 기획했나요?**

동아리·스터디 모집이 종이 게시판이나 메신저로만 이루어져 정보가 흩어지고 신청자 확인도 번거로운 문제를 해결하고자, 모집글 조회부터 등록·신청, 본인 활동 내역 확인까지 한 곳에서 가능한 웹 게시판을 기획했다.

**제한된 시간 내에 우리 팀이 반드시 구현하고자 했던 '핵심 목표'는 무엇인가요?**

10시간 내에 모든 모집글을 한눈에 볼 수 있는 메인 페이지, 모집글을 직접 등록하고 신청할 수 있는 신청·등록 페이지, 본인이 작성·신청한 내역을 확인하는 프로필 페이지, 이렇게 3개 페이지의 핵심 기능을 빠짐없이 구현하는 것을 목표로 했다. 페이지 간 데이터가 끊김 없이 연결되는 것(모집글 등록 → 메인에 반영 → 프로필에서 확인)을 최우선으로 삼았다.

### 역할 분담

| 팀원 | 맡은 세부 업무 |
| --- | --- |
| 김승우 | 프로젝트 초기 세팅(React + TypeScript + Vite), 공통 타입(`types.ts`) 정의, 더미 데이터 구축, 좌측 사이드바·상단 탑바 등 공통 컴포넌트 |

| ____(본인) | **프로필 페이지(`ProfilePage`) 구현** — 본인이 작성한 모집글 / 신청한 모집글 필터링·표시, 활동 통계, App과의 데이터 연동 |

> ※ 본인이 구현한 기능에 대해 직접 발표해야 하므로, 특정 팀원을 '발표 담당' 역할로 나누지 않았다. 각자 맡은 페이지를 본인이 직접 발표한다.

### 시스템 아키텍처 및 데이터 흐름도

```
                    ┌─────────────────────────────────────────────┐
                    │                  App.tsx                     │
                    │   (상태 관리 컨테이너 / 単일 진실 공급원)      │
                    │                                              │
                    │   posts: Post[]   ← 모든 모집글 상태 배열     │
                    │   currentPage     ← 현재 보고 있는 탭         │
                    │   handleAddPost() / handleApply()            │
                    └───────┬───────────────┬───────────────┬──────┘
              props 전달 │               │               │
        ┌─────────────────▼──┐  ┌─────────▼────────┐  ┌──▼──────────────┐
        │   MainPage         │  │  RegisterPage    │  │  ProfilePage     │
        │  (모집글 목록 조회) │  │ (등록·신청)       │  │ (본인 활동 내역) │ ← 내 담당
        │                    │  │                  │  │                  │
        │  posts 읽기 전용    │  │ onAddPost 호출    │  │ posts 필터링      │
        └────────────────────┘  │ onApply 호출      │  │  - author 일치   │
                                └──────────────────┘  │  - applicants 포함│
                                                       └──────────────────┘

데이터 흐름(단방향):
  사용자 입력 → RegisterPage가 App의 함수 호출 → App의 posts 상태 변경
            → React 리렌더 → MainPage·ProfilePage에 자동 반영
```

핵심은 **상태(posts)를 App.tsx 한 곳에서만 관리(상태 끌어올리기, lifting state up)** 하고, 각 페이지는 `props`로 데이터를 받아 화면 표시에만 집중하는 단방향 데이터 흐름 구조다. 덕분에 등록 페이지에서 글을 올리면 메인과 프로필에 별도 동기화 코드 없이 자동 반영된다.

## 4. 역할별 구현 내역

### 기한 내에 본인의 역할을 어떻게 완수했으며, 어떤 기능을 구현했나요?

내가 맡은 **프로필 페이지**는 "본인이 게시판에서 한 활동(작성·신청 내역)을 한 화면에서 확인"하는 역할이다. 구체적으로 다음 기능을 구현했다.

1. **본인 식별(이름 입력) 기능**
   - 이 프로젝트에는 로그인 기능이 없어, 누가 '나'인지 식별할 방법이 없었다. 그래서 이름 입력창을 두고, 입력한 이름을 기준으로 활동 내역을 필터링하도록 설계했다.
2. **내가 작성한 모집글 목록**
   - 전체 `posts` 중 `post.author`가 내 이름과 일치하는 글만 골라 카드로 표시. 각 글의 모집 중/마감 상태 배지와 신청 인원도 함께 보여준다.
3. **내가 신청한 모집글 목록**
   - 각 글의 `applicants` 배열에 내 이름이 포함된 글만 골라 표시.
4. **활동 요약 통계**
   - 작성한 모집글 수 / 신청한 모집글 수를 숫자 카드로 한눈에 표시.
5. **프로필 카드 UI**
   - 이름 첫 글자를 딴 아바타와 사용자 정보를 상단에 배치.
6. **App과의 데이터 연동**
   - `App.tsx`의 `profile` 탭에 있던 "개발 진행 중" 임시 화면을 실제 `<ProfilePage posts={posts} />` 컴포넌트로 교체하여, 메인·등록 페이지와 동일한 `posts` 상태를 공유하도록 연결했다.

### 기능 구현 중 발생한 문제 상황과 해결 과정 (트러블슈팅)

- **문제 1 — 로그인이 없어 '나'를 특정할 수 없음**
  - 상황: 활동 내역을 보여주려면 '현재 사용자'가 누구인지 알아야 하는데, 인증 기능이 없었다.
  - 해결: 로그인 시스템을 새로 만들 시간이 없어, `useState`로 '내 이름' 입력값을 관리하고 그 이름으로 `posts`를 필터링하는 방식으로 대체했다. 주석에 "로그인 기능이 생기면 이 입력창 대신 로그인된 사용자 이름을 쓰면 된다"고 남겨, 추후 확장 지점을 명확히 했다.

- **문제 2 — 첫 화면이 비어 보이는 문제**
  - 상황: 이름 입력 전에는 목록이 모두 비어 있어, 발표·시연 시 화면이 휑하게 보였다.
  - 해결: 더미 데이터에 신청자로 들어가 있는 "김승우"를 이름 입력의 기본값으로 설정해, 페이지를 열자마자 신청 내역이 채워져 보이도록 했다.

- **문제 3 — 입력값 앞뒤 공백으로 필터가 어긋남**
  - 상황: 이름 입력 시 실수로 공백이 들어가면 `author === 이름` 비교가 false가 되어 내 글이 안 잡혔다.
  - 해결: `myName.trim()` 결과를 별도 변수(`trimmedName`)로 만들어 모든 필터·표시에 일관되게 사용하도록 정리했다.

## 5. AI 활용 과정

### 어느 단계에서 AI 도구를 활용했나요?

- **초기 뼈대 코드 생성:** 프로필 페이지 컴포넌트(`ProfilePage.tsx`)의 기본 구조(섹션 분할, 필터링 로직, JSX 골격)를 AI(Claude)로 빠르게 초안 생성했다.
- **스타일(CSS) 작성:** 기존 메인 페이지의 카드·배지 디자인과 톤을 맞춘 `ProfilePage.css` 초안을 AI 도움으로 작성했다.
- **연동 작업:** `App.tsx`의 임시 placeholder를 실제 컴포넌트로 교체하는 부분을 AI와 함께 확인했다.
- **검증:** `tsc -b` 타입 체크를 돌려 오류 없이 빌드되는지 확인했다.

### AI 결과물을 그대로 쓰지 않고, 어떻게·왜 직접 수정·보완했나

> AI가 만든 코드를 그대로 제출하지 않고, 우리 팀 요구사항(가독성, 리액트 상태 관리 효율, 기존 코드와의 일관성)에 맞춰 다음과 같이 직접 수정했다.

1. **(왜) 기본 동작이 우리 데이터 구조와 맞지 않아서 → (어떻게) 필터 기준을 직접 교체**
   - AI 초안은 '로그인된 사용자' 같은 존재하지 않는 값을 가정했다. 우리 프로젝트엔 인증이 없으므로, 실제 데이터 구조(`post.author`, `post.applicants[].name`)에 맞게 **이름 문자열 비교 방식으로 필터 로직을 다시 작성**했다. 이렇게 해야 메인·등록 페이지가 쓰는 동일한 `posts` 데이터로 정확히 동작한다.

2. **(왜) 시연 시 빈 화면이 보기 안 좋아서 → (어떻게) 기본값을 더미 데이터와 연결**
   - 빈 입력 상태의 UX가 좋지 않아, 입력 기본값을 더미 데이터의 실제 신청자("김승우")로 지정했다. AI는 보통 빈 문자열을 기본값으로 두지만, 우리 팀의 더미 데이터를 알고 있는 사람만 할 수 있는 수정이라 직접 바꿨다.

3. **(왜) 입력 공백 버그를 막기 위해 → (어떻게) trim 처리 일원화**
   - 사용자가 공백을 넣어도 필터가 깨지지 않도록 `trimmedName` 변수로 한 번만 가공해 전 구간에서 재사용하도록 리팩터링했다. AI 초안은 곳곳에서 `myName`을 그대로 비교해 버그 소지가 있었다.

4. **(왜) 기존 코드와 디자인·네이밍을 통일하려고 → (어떻게) 스타일 토큰·클래스명 정리**
   - 색상·배지 모양을 메인 페이지(`MainPage.css`)의 `badge-recruiting`/`badge-full`과 동일한 색(#e6f4ea / #fce8e6)으로 맞추고, 클래스명도 `profile-` 접두사로 통일해 다른 팀원이 봐도 어느 페이지 스타일인지 바로 알 수 있게 했다.

5. **(왜) 발표 때 코드를 설명할 수 있어야 해서 → (어떻게) 한글 주석 보강**
   - 필터 로직과 기본값 설정 의도, 추후 로그인 확장 지점을 한글 주석으로 상세히 달아, 코드를 내가 완전히 이해하고 발표할 수 있도록 다듬었다.

## 6. KPT 프로젝트 리뷰

- **Keep (계속 유지할 점)**
  - 상태를 App 한 곳에서 관리하고 각 페이지는 props로 받아 표시만 하는 **단방향 데이터 흐름 구조**. 페이지가 늘어도 데이터 동기화 버그가 없어 협업에 유리했다.
  - 작업 전 팀원과 데이터 구조(`types.ts`)를 먼저 합의하고 시작한 점. 덕분에 내 프로필 페이지가 남이 만든 `posts`와 바로 맞물렸다.
  - AI 초안을 받되 반드시 직접 읽고 수정해 내 것으로 만든 습관.

- **Problem (아쉬웠던 점)**
  - 로그인·인증이 없어 '본인'을 이름 입력으로 우회한 점은 기능적으로 한계가 분명했다(동명이인 구분 불가 등).
  - 데이터가 브라우저 새로고침 시 사라지는 메모리 상태라, 실제 게시판처럼 영구 저장되지 않았다.
  - 초반에 폴더 구조(`pages` vs `screen`)가 통일되지 않아 파일 위치를 헷갈린 적이 있었다.

- **Try (새로 시도해볼 점)**
  - `localStorage` 또는 간단한 백엔드(예: Firebase)를 붙여 **데이터 영구 저장**을 시도한다.
  - 간단한 로그인(이름 선택/저장)을 도입해 '내 활동'을 입력 없이 자동으로 보여준다.
  - 폴더·파일 네이밍 규칙을 작업 시작 전에 문서로 합의하고 시작한다.

### 핵심 소스 코드 — `src/pages/ProfilePage.tsx` (본인 작성·수정)

```tsx
import { useState } from "react";
import "./ProfilePage.css";
import type { Post } from "../types";

interface ProfilePageProps {
  posts: Post[]; // App(부모)이 들고 있는 전체 모집글 상태 배열
}

// 프로필 페이지.
// 이 앱에는 아직 로그인 기능이 없기 때문에,
// "내 이름"을 입력하면 그 이름을 기준으로
// 1) 내가 작성한 모집글  2) 내가 신청한 모집글 을 모아서 보여준다.
// (로그인 기능이 생기면 이 입력창 대신 로그인된 사용자 이름을 쓰면 된다.)
function ProfilePage({ posts }: ProfilePageProps) {
  // 더미 데이터에 신청자로 들어가 있는 "김승우"를 기본값으로 둬서
  // 첫 화면이 비어 보이지 않게 했다.
  const [myName, setMyName] = useState("김승우");

  const trimmedName = myName.trim();

  // 내가 작성한 모집글 (author가 내 이름과 일치)
  const myPosts = posts.filter((post) => post.author === trimmedName);

  // 내가 신청한 모집글 (applicants 안에 내 이름이 포함)
  const appliedPosts = posts.filter((post) =>
    post.applicants.some((a) => a.name === trimmedName)
  );

  // 프로필 아바타에 표시할 이름 첫 글자
  const initial = trimmedName ? trimmedName.charAt(0) : "?";

  return (
    <div className="profile-page">
      {/* 상단 프로필 카드 */}
      <section className="profile-header">
        <div className="profile-avatar">{initial}</div>
        <div className="profile-header-info">
          <h2>{trimmedName || "이름을 입력해주세요"}</h2>
          <p>동아리·스터디 모집 게시판 회원</p>
        </div>
      </section>

      {/* 내 이름 입력 (로그인 대체) */}
      <section className="profile-name-section">
        <label htmlFor="profile-name-input">내 이름</label>
        <input
          id="profile-name-input"
          type="text"
          placeholder="이름을 입력하면 내 활동이 보여요"
          value={myName}
          onChange={(e) => setMyName(e.target.value)}
        />
      </section>

      {/* 활동 요약 통계 */}
      <section className="profile-stats">
        <div className="profile-stat-card">
          <span className="profile-stat-number">{myPosts.length}</span>
          <span className="profile-stat-label">작성한 모집글</span>
        </div>
        <div className="profile-stat-card">
          <span className="profile-stat-number">{appliedPosts.length}</span>
          <span className="profile-stat-label">신청한 모집글</span>
        </div>
      </section>

      {/* 내가 작성한 모집글 목록 */}
      <section className="profile-list-section">
        <h3>✍️ 내가 작성한 모집글</h3>
        {myPosts.length === 0 ? (
          <p className="profile-empty">아직 작성한 모집글이 없어요.</p>
        ) : (
          <ul className="profile-post-list">
            {myPosts.map((post) => {
              const isFull = post.applicants.length >= post.capacity;
              return (
                <li key={post.id} className="profile-post-card">
                  <div className="profile-post-top">
                    <h4>{post.title}</h4>
                    <span
                      className={`profile-badge ${
                        isFull ? "profile-badge-full" : "profile-badge-open"
                      }`}
                    >
                      {isFull ? "모집 마감" : "모집 중"}
                    </span>
                  </div>
                  <p className="profile-post-content">{post.content}</p>
                  <span className="profile-post-meta">
                    신청 {post.applicants.length} / {post.capacity}명
                  </span>
                </li>
              );
            })}
          </ul>
        )}
      </section>

      {/* 내가 신청한 모집글 목록 */}
      <section className="profile-list-section">
        <h3>🙋 내가 신청한 모집글</h3>
        {appliedPosts.length === 0 ? (
          <p className="profile-empty">아직 신청한 모집글이 없어요.</p>
        ) : (
          <ul className="profile-post-list">
            {appliedPosts.map((post) => (
              <li key={post.id} className="profile-post-card">
                <div className="profile-post-top">
                  <h4>{post.title}</h4>
                  <span className="profile-post-meta">작성자: {post.author}</span>
                </div>
                <p className="profile-post-content">{post.content}</p>
              </li>
            ))}
          </ul>
        )}
      </section>
    </div>
  );
}

export default ProfilePage;
```

> `App.tsx` 연동 부분(본인 수정): `profile` 탭의 임시 placeholder를 `return <ProfilePage posts={posts} />;` 로 교체하여 전체 상태와 연결.
